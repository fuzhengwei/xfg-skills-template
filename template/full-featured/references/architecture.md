# 架构设计文档

> 当需要理解系统整体设计时读取此文档。

## 系统架构

```
┌──────────────┐     ┌──────────────┐     ┌──────────────┐
│   输入验证    │────▶│   数据处理    │────▶│   输出验证    │
│  validate.sh │     │  process.py  │     │  process.py  │
└──────────────┘     └──────────────┘     └──────────────┘
       │                    │                    │
       ▼                    ▼                    ▼
   输入文件            中间结果              最终输出
```

## 数据流

1. **输入阶段**：验证脚本检查文件格式、编码和完整性
2. **处理阶段**：处理脚本执行核心业务逻辑
3. **输出阶段**：验证输出结果的正确性和完整性

## 配置

使用 `assets/template.yaml` 作为配置模板。
